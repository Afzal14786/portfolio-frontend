This repository contains the **Public-Facing Client Application** of the Integrated Portfolio & Blog Management System. It is built to serve dynamic content fetched from the custom backend API, offering a seamless, highly optimized, and visually stunning user experience.

---

## 🎯 Key Features

- **Dynamic Server-Driven UI:** All data (Projects, Skills, Journey, Certificates, Blogs, and Profile) is fetched dynamically from the backend API. No hardcoded content.
- **macOS Glassmorphism Design:** Implements a modern UI paradigm using frosted glass effects (`backdrop-blur`), subtle glowing borders, and floating elements via Tailwind CSS.
- **Strict Type Safety:** 100% TypeScript implementation with rigorous interfaces matching the backend schema to prevent runtime errors.
- **Scroll-Spy Navigation:** The fixed header mathematically calculates viewport intersection to dynamically highlight the currently active section as the user scrolls.
- **Bento Grid Architecture:** Uses an asymmetrical, responsive grid layout for the `/me` profile page to elegantly display varied data types (Quotes, Hobbies, Links).
- **Interactive Modals & Lightboxes:** Features a custom-built, full-screen image Lightbox for viewing Certificates and detailed Modals for Projects/Skills.
- **Markdown/HTML Rendering:** Safely parses and injects rich-text HTML content generated by the Admin Dashboard's WYSIWYG editor into the public blog reader.

## 📂 Project Architecture

The application follows a highly modular **Separation of Concerns** architecture, making it enterprise-ready and easily maintainable.

```text
src/
├── api/                  # Axios instance configuration & interceptors
├── assets/               # Static assets (Original brand icons, GIFs)
├── components/           # Reusable UI building blocks
│   ├── cards/            # Standardized cards (Blog, Project, Skill, etc.)
│   ├── layout/           # Global wrappers (Header, Footer, Main Layout)
│   └── sections/         # Heavy page sections (Hero, Journey, Connect)
├── lib/                  # Helper functions and core configurations
├── Pages/                # Top-level Route components (Home, AllBlogs, Profile)
├── services/             # API Abstraction Layer (blog.service.ts, profile.service.ts)
├── types/                # Strict TypeScript Interfaces matching backend Schemas
├── App.tsx               # Main application router & global states
└── main.tsx              # React DOM mounting and initialization
```  

## ⚙️ Core Services & API Integration
The frontend abstracts all backend communication into dedicated `Service` classes within the `  ` directory. This ensures UI components only handle rendering, not data fetching logic.  

*   `ProfileService`: Fetches dynamic Hero data, Hobbies, Social Links, and Reading Resources.

* `BlogService`: Fetches paginated lists of published blogs, individual blogs via slug, and related articles.

* `CertificateService`: Retrieves authenticated certification data including Issuer, Skills, and Cloudinary URLs.  

*(All API responses are wrapped in strict TypeScript interfaces located in `src/types/` to guarantee data integrity before rendering).*  

## 💻 Local Setup & Installation  

**Prerequisites**  

* Node.js (v16+ recommended)
* npm or yarn  

**1. Clone the repository**  

```bash
git clone https://github.com/Afzal14786/portfolio-frontend.git  
cd portfolio-frontend  
```  

**2. Install dependencies**  
```bash
npm install
```  

**3. Set up Environment Variables**  
Create a `.env` file in the root directory and connect it to your local or deployed Backend API.
```text
# Example .env configuration
VITE_API_BASE_URL=http://localhost:5000/api/v1
```  

**4. Run the Development Server**  
```bash
npm run dev
```
*The application will be available at `http://localhost:5173`.*  

## 🎨 Design System  

The application relies heavily on utility classes to achieve the Glassmorphism look.
A standard card component utilizes the following Tailwind pattern:  

`bg-white/60 backdrop-blur-xl rounded-[2.5rem] border border-white/80 shadow-[0_8px_30px_rgb(0,0,0,0.04)]`  

This creates a semi-transparent, blurred background with a crisp white border, allowing background elements to softly bleed through the interface.
